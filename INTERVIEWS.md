# 🎤 Technical Interviews Guide (Greek Tech Market)

Ο τεχνικός έλεγχος στις ελληνικές εταιρείες (Banking, Telcos, Startups, Consulting) συνήθως περιλαμβάνει τα εξής στάδια:

## 1. Το Πρώτο Στάδιο: HR Screening
Απλές ερωτήσεις για το background, το motivation, και τις μισθολογικές προσδοκίες. Εδώ εστιάζουν κυρίως στο "Culture Fit".
- *Συμβουλή:* Να είστε ξεκάθαροι για το αν ψάχνετε Remote, Hybrid, ή On-site. 

## 2. Το Τεχνικό Τεστ (Live Coding ή Platform)
Συνήθως γίνεται μέσω πλατφορμών όπως το **HackerRank** ή **Codility**.
- **Για Data Analysts / Engineers:** Θα αντιμετωπίσετε σίγουρα **SQL**. Θα σας ζητηθεί να κάνετε JOINs, Window Functions (`ROW_NUMBER()`, `RANK()`), και Data Aggregation.
- **Για Data Scientists:** Ερωτήσεις σε Python (π.χ. data manipulation με Pandas), βασικοί αλγόριθμοι, και ίσως ερωτήσεις Statistics / Probability.

## 3. Το Take-Home Project (Προαιρετικό αλλά σύνηθες)
Πολλές εταιρείες δίνουν ένα μικρό dataset (συχνά ανώνυμα πραγματικά δεδομένα τους) και σας ζητούν:
- **Analyst:** Να φτιάξετε ένα Dashboard (Power BI / Tableau) ή ένα report με business insights.
- **Scientist:** Να φτιάξετε ένα Predictive Model (π.χ. churn prediction) ή ένα RAG system, και να παραδώσετε το Jupyter Notebook ή τον Python κώδικα.
- *Η παγίδα:* Δεν ψάχνουν απλώς να βρείτε το σωστό νούμερο. Ψάχνουν να δουν **πώς σκέφτεστε**, πώς καθαρίζετε τα δεδομένα (EDA), και αν γράφετε καθαρό κώδικα (Clean Code, comments).

## 4. Το Τελικό Στάδιο (Technical / Business Interview)
Παρουσίαση του project σας μπροστά στην ομάδα ή τον/την Head of Data.
- Εδώ πρέπει να δείξετε το **Data Storytelling** σας. 
- *Κλασική ερώτηση:* "Πώς θα εξηγούσες αυτό το μοντέλο στον CEO που δεν ξέρει από Data Science;"

## 💡 Pro Tips για την Ελλάδα
1. **Δείξτε Business Acumen:** Στην Ελλάδα, πολλές ομάδες Data είναι μικρές. Οι εταιρείες ψάχνουν άτομα που δεν γράφουν απλά κώδικα, αλλά καταλαβαίνουν τι φέρνει χρήματα (ROI) στην εταιρεία.
2. **Αγγλικά:** Παρότι είμαστε στην Ελλάδα, τα interview συχνά γίνονται μερικώς στα Αγγλικά, καθώς πολλές ομάδες είναι πολυεθνικές.

## 5. Behavioral & Cultural Fit (Η Μέθοδος STAR)
Πολλοί υποψήφιοι "κόβονται" ενώ έχουν τέλειο κώδικα, επειδή δεν ταιριάζουν στην κουλτούρα της ομάδας. Στην Ελλάδα, οι εταιρείες εκτιμούν το "ownership" (πρωτοβουλία) και την ανθεκτικότητα σε συνθήκες πίεσης ή legacy systems.

Θα σας ρωτήσουν: *"Πες μου για μια φορά που διαφώνησες με έναν συνάδελφο ή manager για τον τρόπο υλοποίησης."*

Χρησιμοποιήστε τη μέθοδο **STAR**:
- **S (Situation):** "Στην προηγούμενη δουλειά, είχαμε ένα dataset με 10 εκατ. γραμμές."
- **T (Task):** "Έπρεπε να παραδώσω το report σε 2 μέρες, αλλά η βάση αργούσε (legacy σύστημα)."
- **A (Action):** "Ο manager πρότεινε να το τρέξουμε το βράδυ. Εγώ διαφώνησα τεκμηριωμένα, πρότεινα να χρησιμοποιήσουμε Apache Spark (ή Polars), έφτιαξα ένα γρήγορο Proof of Concept σε 2 ώρες και του το έδειξα."
- **R (Result):** "Το δέχτηκε, μειώσαμε τον χρόνο εκτέλεσης κατά 80%, και προλάβαμε το deadline."

## 6. Τι ΠΡΕΠΕΙ να ρωτήσετε εσείς τον Interviewer
Όταν σας πουν "Έχετε κάποια ερώτηση για εμάς;", **μην** πείτε "Όχι". Ρωτήστε:
1. *"Ποιο είναι το μεγαλύτερο Data Quality πρόβλημα που αντιμετωπίζει η ομάδα σας σήμερα;"* (Δείχνει ωριμότητα).
2. *"Πώς μετράτε την επιτυχία αυτού του ρόλου; Ποια είναι τα KPIs στους πρώτους 6 μήνες;"* (Δείχνει Business orientation).
3. *"Έχετε αρχίσει να ενσωματώνετε LLMs/Generative AI (π.χ. Copilot, LangChain) στα εσωτερικά σας workflows;"* (Δείχνει ότι σκέφτεστε την "Agentic Era").

## 7. System Design & Architecture (Για Mid/Senior Ρόλους)
Αν διεκδικείς θέση Data Engineer ή Senior ML Engineer, θα σου δώσουν έναν λευκό πίνακα (whiteboard) ή ένα ψηφιακό εργαλείο (π.χ. Miro/Excalidraw) και θα σου ζητήσουν: *"Σχεδίασε ένα σύστημα που παίρνει real-time δεδομένα από 1.000 καταστήματα λιανικής και βγάζει personalized recommendations στο κινητό του πελάτη."*
- **ΛΑΘΟΣ:** Να ξεκινήσεις να μιλάς κατευθείαν για τον αλγόριθμο (π.χ. "Θα βάλω ένα Deep Neural Network").
- **ΣΩΣΤΟ:** Να μιλήσεις για Data Pipeline.
  - Ξεκίνα από το **Ingestion**: (π.χ. Apache Kafka για streaming data).
  - Πήγαινε στο **Storage**: (π.χ. Data Lake σε AWS S3 ή Azure Data Lake).
  - Συνέχισε στο **Compute/Processing**: (π.χ. Spark Streaming / Databricks για τον καθαρισμό).
  - Κατέληξε στο **Serving**: (π.χ. ένα γρήγορο API σε FastAPI ή μια Vector DB για caching).

## 8. Red Flags: Πώς να καταλάβετε αν η Εταιρεία είναι Τοξική
Η συνέντευξη είναι δρόμος **δύο κατευθύνσεων**. Εσείς επιλέγετε τον εργοδότη, όσο σας επιλέγει κι αυτός. Αν ακούσετε τα παρακάτω, ίσως πρέπει να απορρίψετε την προσφορά:
- 🚩 *"Δεν έχουμε Data Engineers, οπότε οι Data Scientists τα κάνουν όλα."* 
  > *(Μετάφραση: Θα περνάς το 95% του χρόνου σου φτιάχνοντας pipelines και καθαρίζοντας σπασμένα CSV, και 5% φτιάχνοντας μοντέλα).*
- 🚩 *"Θέλουμε κάποιον να μας βάλει AI σε όλα αύριο, τώρα που είναι της μόδας."* 
  > *(Η ηγεσία έχει πέσει θύμα του hype. Δεν καταλαβαίνουν ότι το AI χρειάζεται Data Maturity. Θα αποτύχεις γιατί θα σου ζητούν μαγικά).*
- 🚩 *"Τα τελικά reports πηγαίνουν στη διοίκηση σε Excel μέσω email."* 
  > *(Αν είμαστε στην Agentic Era και δεν έχουν έστω ένα αυτοματοποιημένο Power BI dashboard, η κουλτούρα της εταιρείας είναι κολλημένη στο 2010).*
