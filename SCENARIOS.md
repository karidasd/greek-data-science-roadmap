# 🏢 Day in the Life: Real Greek Business Cases

Στα interviews ή στις πρώτες μέρες στη δουλειά, δεν θα σας δώσουν το "Titanic Dataset". Θα σας δώσουν πραγματικά προβλήματα που κοστίζουν (ή βγάζουν) λεφτά στην εταιρεία. Εδώ είναι 3 κλασικά σενάρια από την Ελληνική αγορά:

## 1. 🏦 Banking: Loan Default Prediction
**Ο κλάδος:** Συστημικές Τράπεζες (Eurobank, Εθνική, Πειραιώς, Alpha)
**Το πρόβλημα:** "Έχουμε 50,000 πελάτες που ζήτησαν καταναλωτικό δάνειο φέτος. Ποιοι από αυτούς δεν θα πληρώσουν τις δόσεις τους (default) στους επόμενους 12 μήνες;"
**Τι Data θα έχετε:** 
- Δημογραφικά (ηλικία, επάγγελμα)
- Credit History (Τειρεσίας, ιστορικό καθυστερήσεων)
- Transactional Data (πόσα ξοδεύουν από την κάρτα τους)
**Πώς σκέφτεται ο Senior:** 
Δεν τον νοιάζει να τρέξει ένα τέλειο XGBoost με 99% accuracy. Τον νοιάζει το **Recall vs Precision**. Αν το μοντέλο πει "δώσε δάνειο" σε κάποιον που τελικά δεν πληρώσει, η τράπεζα χάνει χιλιάδες ευρώ. Η συνέπεια των False Positives είναι τεράστια.

## 2. 🛒 E-Commerce: Market Basket Analysis & Recommendations
**Ο κλάδος:** Μεγάλα Marketplaces (Skroutz, efood, Public)
**Το πρόβλημα:** "Όταν ένας χρήστης βάζει ένα iPhone 15 στο καλάθι του, τι άλλο πρέπει να του προτείνουμε για να αυξήσουμε το Average Order Value (AOV);"
**Τι Data θα έχετε:** 
- Εκατομμύρια γραμμές από logs (clickstream)
- User IDs, Product IDs, Timestamps, Categories
**Πώς σκέφτεται ο Senior:** 
Η ταχύτητα μετράει. Το μοντέλο (είτε RAG system, είτε Collaborative Filtering) πρέπει να δίνει recommendations σε milliseconds. Επίσης, πρέπει να γίνει A/B Test στο site για να αποδειχθεί ότι το μοντέλο όντως αύξησε τον τζίρο σε σχέση με το να δείχναμε απλώς τα "Top Selling" προϊόντα.

## 3. 📉 NPLs (Non-Performing Loans): Servicer Portfolio Analysis
**Ο κλάδος:** Εταιρείες Διαχείρισης Απαιτήσεων (doValue, Cepal, Intrum) - Τεράστιος κλάδος στην Ελλάδα την τελευταία δεκαετία.
**Το πρόβλημα:** "Έχουμε αγοράσει ένα χαρτοφυλάκιο (portfolio) με 10,000 κόκκινα δάνεια αξίας 1 δισ. ευρώ. Σε ποιους οφειλέτες πρέπει να τηλεφωνήσει το call center αύριο, ώστε να μεγιστοποιήσουμε τις εισπράξεις;"
**Τι Data θα έχετε:** 
- Ύψος οφειλής
- Εξασφαλίσεις (π.χ. αξία ακινήτου)
- Ιστορικό κλήσεων και ρυθμίσεων
**Πώς σκέφτεται ο Senior:** 
Είναι πρόβλημα Optimization και Segmentation. Θα χωρίσει τους οφειλέτες σε buckets (π.χ. "High Will / Low Capacity"). Ένα AI-Augmented dashboard θα προτείνει στην ομάδα operations ποια είναι η βέλτιστη ρύθμιση για τον κάθε οφειλέτη.
